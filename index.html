Player

A player represents a connection between Discord and Lavalink

A player is the primary way to interact with Lavalink

new Player(node: LavalinkNode, guild: String, shard: module:eris.Shard)

Extends EventEmitter

Parameters
node (LavalinkNode) The node supporting this player
guild (String) The ID of the guild where this player is used
shard (module:eris.Shard) The shard that runs this player
Properties
node (LavalinkNode) : The node supporting this player
guild (String) : The ID of the guild where this player is used
shard (module:eris.Shard) : The shard that runs this player
paused (Boolean) : Whether or not the playback for this player is paused
connected (Boolean) : Whether or not the player is fully connected
state (Object) : The current state of the player
  • state.time Number?

    The UNIX timestamp when this player started playing

  • state.position Number?

    A time in ms representing the position of the track the player is currently playing

Player#trackEnd

This will be fired whenever a track stops playing

A track can end for many reasons! You should check the reason property before doing additional logic, like playing new tracks

Player#trackEnd

Type: Object

Properties
track (String) : The track that has ended
reason (String) : Why the track ended
Example
Player.on('trackEnd', ctx => console.log(`The track ${ctx.track} has ended`))

Player#trackError

This will be fired whenever lavalink encounters an error while playing a track

Player#trackError

Type: Object

Properties
track (String) : The track that has ended
error (String) : Why the track errored
Example
Player.on('trackError', ctx => console.error(`The track ${ctx.track} crashed!`, ctx.error))
